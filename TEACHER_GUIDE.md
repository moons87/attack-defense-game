# 👨‍🏫 Руководство учителя - Как провести урок

## 📋 Этап 1: Подготовка (15 минут до урока)

### 1.1 Настройка Firebase (один раз)
- Перейди на [console.firebase.google.com](https://console.firebase.google.com)
- Создай новый проект (например, "cybersecurity-game")
- Перейди в "Realtime Database" и нажми "Create Database"
- Выбери регион (closest to you)
- Скопируй credentials и добавь в `.env.local`:

```
NEXT_PUBLIC_FIREBASE_API_KEY=AIza...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_DATABASE_URL=https://project.firebaseio.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=project-id
```

### 1.2 Запуск приложения
```bash
npm install
npm run dev
# или на Vercel - просто откройте ссылку
```

### 1.3 Подключение к проектору
- Откройте на своем компьютере: http://localhost:3000 (или Vercel URL)
- Подключите проектор
- Создайте новую комнату - система выдаст **код комнаты** (5 символов)
- Выберите роль: **Admin**

## 🎮 Этап 2: Начало урока (5 минут)

### 2.1 Рассказ о игре
- "Сейчас мы будем играть в кибербезопасность!"
- "Красная команда - атакующие (hackers)"
- "Синяя команда - защитники (security)"

### 2.2 Раздача кодов
- Покажите **код комнаты** на экране
- Попросите учеников перейти на **http://localhost:3000** (или ваш Vercel URL)
- Каждый вводит код и выбирает роль: Red (Атакующий) или Blue (Защитник)

## ⚙️ Этап 3: Ход игры (20-30 минут)

### Один раунд длится ~1-2 минуты:

#### Шаг 1: Объяснение (30 сек)
Объясни ученикам, что происходит в этом раунде:
- "Red Team: выберите атаку - скажите, как вы будете атаковать систему?"
- "Blue Team: выберите защиту - как вы защититесь?"

#### Шаг 2: Голосование (30 сек)
- Ученики **выбирают** атаку/защиту
- Нажимают **"Отправить голос"**

Примерный диалог:
- "Red Team выбирает между: Фишинг, Подбор пароля, Уязвимость, Доступ к данным"
- "Blue Team выбирает между: Обучение, Аутентификация, Обновления, Контроль доступа"

#### Шаг 3: Разрешение (30 сек)
- **ВЫ** (Admin) видите количество голосов
- Нажимаете **"▶️ Подвести итог раунда"**
- Система определяет:
  - Кто выиграл голосование в красной команде (обычно большинство)
  - Кто выиграл голосование в синей команде
  - Вероятность успеха атаки
  - Результат: ✅ УСПЕХ или 🛡️ ЗАЩИТА

#### Шаг 4: Разбор (1 мин)
- Объясните результат
- "Почему это произошло?"
- "Что можно было сделать лучше?"

**Пример раунда:**
```
Red Team выбрала: Фишинг (strength 0.6)
Blue Team выбрала: Обучение (strength 0.7)

Оба относятся к слою "Сотрудники"
Вероятность успеха: 0.6 - 0.7 = -0.1 (максимум 0)
Результат: 🛡️ ЗАЩИТА (Blue Team защитилась)
Blue Team получает 1 очко!
```

## 💡 Методические советы

### Делайте после каждого раунда:
1. **Обсуждение** - "Почему Red выбрала эту атаку?"
2. **Объяснение** - "Фишинг работает на людей, поэтому..."
3. **Тактика** - "Может быть, Red должна была выбрать Уязвимость?"

### Вариации игры:

**Игра 1: Обучение атакам (10 раундов)**
- Red Team изучает разные атаки
- Blue Team использует одну защиту

**Игра 2: Обучение защитам (10 раундов)**
- Blue Team изучает разные защиты
- Red Team использует разные атаки

**Игра 3: Стратегия (15+ раундов)**
- Обе команды выбирают стратегию
- "Red Team: какая атака наиболее вероятна против синего?"
- "Blue Team: предугадайте атаку!"

## 📊 Финальная статистика

После игры:
- **Показать счет**: "🔴 Red: X | 🔵 Blue: X"
- **Объяснить**: "Что сработало лучше?"
- **Домашнее задание**: "Как бы вы изменили правила?"

## 🔍 Отладка

### Проблема: Ученики не видят обновления
**Решение**: 
- Обновите страницу (F5)
- Проверьте, что вы нажали "Отправить голос"

### Проблема: Firebase ошибка
**Решение**:
- Проверьте `.env.local`
- Убедитесь, что Database URL содержит `.firebaseio.com`
- Перезагрузите сервер

### Проблема: Коды комнаты не совпадают
**Решение**:
- Используйте 5-значный код со страницы админа
- Код должен остаться одинаковым для всех

- ✅ Основные типы атак

- ✅ Как защита противостоит атаке

- ✅ Командную работу в кибербезопасности

- **Игра**: 20-40 минут (можно сделать 10-20 раундов)
- **Разбор**: 10 минут

**Итого: 50-70 минут (один урок)**
